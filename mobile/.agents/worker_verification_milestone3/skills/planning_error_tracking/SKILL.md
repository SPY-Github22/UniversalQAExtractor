# Planning & Error Tracking Skill Copy

## Responsibilities
- Create a clear, step-by-step implementation plan before executing any task.
- Consult the user with the plan and wait for approval.
- Maintain an accurate log of all encountered errors and mistakes in `errors_faced.txt`.
- Document working solutions alongside the errors once an error is successfully resolved, treating the file as a knowledge base.
- Track milestones and ensure all prerequisites are met.

## Instructions
1. When assigned a task, write down a plan detailing each step.
2. Do not proceed with execution until the user has reviewed and approved the plan.
3. If an error occurs at any point, append it to `errors_faced.txt` in the workspace root. Include the timestamp, error message, and context.
4. Once an error is successfully resolved, you MUST update `errors_faced.txt` to include the working solutions alongside the errors so that the file acts as a knowledge base.
5. Review the error log periodically to avoid repeating mistakes.
